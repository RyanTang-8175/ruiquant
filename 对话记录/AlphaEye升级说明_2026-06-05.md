# AlphaEye 升级说明（2026-06-05）

本轮升级目标：基于现有 AlphaEye 项目继续升级，不推翻重来；把 DeepSeek 从“普通聊天模型”约束为金融研究 Agent 的推理层，把数据、工具、记忆、验证、风险闸门和审计记录补齐。

## 一、金融 AI 底座

新增数据源注册中心：

- `src/data/providers/registry.py`
- `src/data/providers/open_provider.py`
- `src/data/providers/ifind_provider.py`

作用：

- 当前继续使用公开数据源。
- iFinD API 到位后只需补 `IFindProvider` 的真实接口映射。
- 上层 AI、雷达、实验室、页面不需要重写。
- “我的”页可切换 `open / ifind` 数据源并保存 iFinD 用户名、Token。

## 二、AI 不再只是 DeepSeek 聊天框

升级点：

- `src/ai/chat.py` 接入数据库会话记忆。
- 用户消息和 AI 回复写入 `ai_sessions / ai_messages`。
- 对个股分析写入 `ai_analysis_records`。
- 每次 AI 工具调用写入 `data/scratchpad/*.jsonl` 审计日志。
- `src/ai/prompts.py` 增加安全闸门：默认输出观察、模拟、验证、放弃，不输出实盘买入指令。

这意味着 DeepSeek 只是语言模型，AlphaEye 的金融专属能力来自：

- 数据源层
- 工具调用
- 股票记忆
- 实验室验证
- 用户执行风险闸门
- 审计 scratchpad

## 三、AI 研究记忆

新增/增强：

- `ConversationMemory.list_recent_threads`
- `ConversationMemory.get_stock_conversations`
- `ConversationMemory.search_messages`
- AI 页面新增“研究记忆”折叠面板

现在可以：

- 查看最近研究会话。
- 查看当前股票历史对话。
- 按关键词搜索历史。
- 将历史记录带入追问。

这为未来 iOS App 做了接口准备，不再依赖单个 JSON 文件。

## 四、用户风险闸门

新增：

- `src/risk/user_state.py`

作用：

- 根据模拟盘胜率、亏损、实验室反馈判断用户状态。
- 状态包括 `normal / caution / cooldown`。
- 如果出现追高、不止损、连续亏损，AI 只能给观察、模拟和复盘计划。

## 五、短线实验室可用化

重写：

- `src/pages/lab.py`

新增：

- `src/lab/backfill.py`

实验室现在能做：

- 新建研究计划。
- 必填研究假设、触发条件、失效条件、止损规则。
- 待验证列表。
- 手动立即回填。
- 复盘 T+1/T+2/T+3 表现。
- 对具体验证记录提交执行反馈。

雷达页和股票详情页的“加入验证”现在会带完整计划，不再是空记录。

## 六、自动回填

`src/scheduler.py` 已接入实验室回填：

- 原预测回填继续保留。

- 使用统一数据源获取日线数据。
- 回填高低收变化、是否触达 +2%、最大回撤等。
