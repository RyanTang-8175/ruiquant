# AlphaEye iFinD 投研工作台升级方案

## 定位

AlphaEye 不再只是聊天和选股按钮，而是 iFinD 驱动的小型个人投研工作台。

主流程：

```text
发现信息 -> 判断真假热点 -> 生成候选 -> 公司研究 -> 证据化 AI 结论 -> 写入审计 -> T+1/T+2/T+3 回放
```

## 第一阶段边界

本阶段明确不做：

- 自动交易
- 复杂回测系统
- 完整多 Agent 框架
- 黑色炫酷 UI
- 全部 iFinD 接口一次性铺满

## 页面结构

```text
行情
雷达
研究
AI
审计
我的
```

- 行情：指数、榜单、快讯。
- 雷达：信息雷达、智能选股、候选生成、伪热点过滤。
- 研究：单家公司研究底稿、证据评分、情景推演、审计入口。
- AI：保留当前回答风格，但必须走工具、证据和置信度约束。
- 审计：研究假设、T+1/T+2/T+3 回放、命中率。
- 我的：iFinD 配置、数据源状态、调用统计、缓存状态。

## iFinD 月额度治理

所有 iFinD 调用必须经过缓存、指纹和用量统计。

缓存建议：

```text
实时行情：20-60 秒
智能选股：10-30 分钟
公告/研报/财务/宏观：6-24 小时
深度研究底稿：6-48 小时
```

我的页必须展示：

```text
今日调用次数
本月调用次数
各接口调用次数
缓存命中率
最近调用时间
当前数据源状态
```

## Research Harness

Harness 负责执行，AI 负责判断。

职责：

```text
指纹去重
缓存
额度治理
结果裁剪
研究记忆
低质量研究记录
审计入口
```

流程：

```text
用户/AI 发起研究
-> 生成 fingerprint
-> 查缓存
-> 缓存命中直接返回
-> 未命中才调用数据源
-> 明细存本地
-> AI 只看摘要
-> 研究结果写入知识库/审计
```

## iFinD Evidence Score

旧评分保留作参考，新评分作为主线并行验证。

维度：

```text
资金热度分
承接质量分
信息催化分
基本面安全分
拥挤/反量化风险分
数据置信度分
```

输出：

```text
机会分
风险分
数据置信度
行动建议：只观察 / 可观察 / 可模拟验证
证据摘要
旧评分差异
```

## AI 证据闸门

AI 回答必须遵守：

```text
没有实时行情，不能说当前价格。
没有公告/新闻，不能说有利好。
数据冲突，必须提示数据源不一致。
iFinD 不可用，自动降置信度。
核心结论必须能回到证据。
回答必须显示数据时间或数据状态。
```

## MiroFish 借鉴方式

不引入完整 MiroFish，不做重型 swarm simulation。

只借鉴轻量流程：

```text
iFinD 数据 = seed material
证据聚合 = evidence graph
AI 输出 = scenario report
```

公司研究底稿必须包含三情景：

```text
情景 A：利好继续发酵
情景 B：冲高回落
情景 C：板块不联动
```

每个情景包含：

```text
可能性
证据
触发条件
失效条件
观察周期
```

## 第一阶段验收标准

```text
雷达页不再只是新闻，而是多源信号工作台。
输入股票能生成完整研究底稿。
AI 回答能展示证据来源和数据状态。
新旧评分能并行显示。
iFinD 调用有缓存和用量统计。
一条研究可以进入审计并等待回放。
数据不足时 AI 明确降置信度。
```
