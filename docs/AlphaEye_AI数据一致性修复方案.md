# AlphaEye AI 数据一致性修复方案

更新时间：2026-06-06

## 问题

AI 回答中国铝业（601600）时出现以下内容：

- 当前价写成 7.57 元；
- 涨跌幅写成 -1.30%；
- 换手率写成 0.94%；
- 成交额写成 9.7 亿；
- 在用户未说明、交易记录未证明的情况下，补写“周五收盘前还加仓了”。

同一轮 Scratchpad 工具审计实际记录为：

- 最新可用价 10.68 元；
- 涨跌幅 -2.02%；
- 换手率 2.04%；
- 成交额约 29.0 亿；
- 没有“周五收盘前加仓”的可核验记录。

这说明问题不在于完全没有数据，而在于“工具证据正确，模型最终文本没有强制服从证据”。

## 根因

### 1. 提示词是软约束

项目已有实时行情提示词、Research Harness 缓存刷新和旧价格警告，但大模型仍可能复制历史对话或旧研究底稿里的数字。

### 2. 最终答案缺少确定性校验

`AIChat._finalize_response` 之前直接把模型回答写入：

- 页面历史；
- JSON 对话文件；
- 数据库会话；
- 研究审计；
- Scratchpad。

一旦回答含旧价格，这个错误还会进入下一轮上下文，形成重复污染。

### 3. 数据源“配置可用”不等于本次报价来自 iFinD

Provider 健康检查只说明 iFinD token 可用。具体行情请求如果失败，会回退腾讯或新浪公开源。

因此回答必须显示本次报价的真实来源，而不是笼统写“已经接入 iFinD”。

### 4. 用户真实持仓、用户口述和模拟盘持仓混在一起

用户说“我持有一万股”只能标注为“用户本轮口述”。模拟盘记录不等于用户券商真实账户，更不能推断用户在某个时间加仓。

## 修复架构

新增 `src/ai/evidence_guard.py`，在最终回答展示和持久化之前执行硬校验：

```text
用户问题
  -> DeepSeek + 工具调用
  -> 本轮证据快照
     - 实时/最新可用行情
     - 工具行情
     - 模拟盘持仓
     - 可核验交易记录
  -> AnswerEvidenceGuard
     - 当前价校验
     - 涨跌幅校验
     - 换手率校验
     - 成交额校验
     - 持仓来源标注
     - 交易时间事实校验
  -> 历史/数据库/Scratchpad/研究审计
```

## 校验规则

### 当前行情

只修改明确表达“当前事实”的数字：

- 当前价；
- 现价；
- 最新价；
- 收盘价；
- 今日/周五涨跌幅；
- 当前换手率；
- 当前成交额。

不会修改：

- 支撑位；
- 压力位；
- 止损位；
- “如果跌破 7.45 元”这类条件价。
- “如果周一上涨 3%”这类未来情景；
- 明确标注为历史年度、季度或过去区间的价格与成交指标。

### 无行情时

如果 iFinD 和公开源都没有返回可靠行情，模型写出的“当前 7.57 元”会被改为：

`当前价格暂无可靠数据`

不会保留无法验证的数字。

### 持仓

- 用户本轮明确说明：显示“按你本轮口述”；
- 用户未说明、模拟盘有记录：显示“模拟盘记录显示”；
- 两者都没有：删除模型编造的持仓数量；
- 用户真实账户口述优先于模拟盘，二者不能混为一谈。

### 交易行为

“周五加仓”“今天卖出”“刚刚减仓”等陈述必须满足至少一个条件：

- 用户本轮明确说过；
- 本地交易记录存在相同方向、相同日期或相同时段的记录。

只有历史买入记录，不能推断为周五加仓。
用户问“我该不该加仓”也不代表已经加仓，疑问、计划和建议请求不能作为已成交证据。

### 数据来源

每次个股回答顶部增加数据硬校验栏：

- 最新可用价格；
- 数据源实际返回的涨跌幅；
- 数据源实际返回的换手率；
- 数据源实际返回的成交额；
- 实际来源；
- 校验时间；
- 纠正冲突数量。

数据源没有返回的指标不会用 `0` 冒充真实值，而是直接不显示。

来源示例：

- `iFinD`
- `腾讯公开源兜底（iFinD 本次未成功返回该报价）`
- `新浪公开源兜底（iFinD 本次未成功返回该报价）`

## iFinD 额度处理

硬校验不会无条件再次调用 iFinD。

`AIChat` 为每轮对话建立 `_request_evidence`：

- `_live_quote_guard` 第一次取得行情后写入快照；
- `get_stock_quote` 工具结果也写入快照；
- 最终校验优先复用快照；
- 只有本轮完全没有行情证据时才补查一次。

这样既保证最终答案有数据依据，也避免同一轮重复消耗 iFinD 月额度。

## 修复后的示例

错误回答：

> 当前7.57元，周五跌-1.30%，换手0.94%，成交9.7亿。你持有1万股，周五收盘前还加仓了。

修复后：

> 数据硬校验：中国铝业(601600) 最新可用报价 10.68元，涨跌幅-2.02%，换手率2.04%，成交额29.0亿；来源：iFinD。
>
> 当前10.68元，周五涨跌幅-2.02%，换手率2.04%，成交额29.0亿。按你本轮口述，你持有1万股；关于是否在周五收盘前加仓，当前没有可核验记录。

## 验收标准

- 模型旧价格不能进入最终历史和数据库；
- 无行情时不能保留模型编造的当前价；
- 条件价不能被误改；
- 用户口述与模拟盘必须明确区分；
- 交易行为必须通过时间匹配；
- 最终答案显示本次报价真实来源；
- 本轮已取行情必须复用，不额外消耗 iFinD；
- 专项测试、全量测试和 Streamlit 冒烟全部通过。
